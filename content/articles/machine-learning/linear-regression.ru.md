Линейная регрессия — самая простая модель, и именно поэтому она важна: на ней проще всего увидеть, что _значит_ «обучить модель».

## Постановка задачи

У вас есть пары $(x, y)$ — например, размер квартиры и её цена. Вы хотите по новому $x$ предсказать $y$. Самое простое предположение: между ними прямая линия.

$$\hat{y} = w \cdot x + b$$

Два параметра: $w$ (наклон) и $b$ (сдвиг). Задача обучения — найти такие $w$ и $b$, чтобы линия проходила «поближе» ко всем точкам.

## Что значит «поближе»

Нужна **функция ошибки**. Самая распространённая — средний квадрат отклонения (MSE):

$$L(w, b) = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$

Чем меньше $L$, тем лучше линия описывает данные. Квадрат используется потому, что:

- он всегда положителен (отклонения вверх и вниз не уничтожают друг друга);
- большие ошибки штрафуются непропорционально сильно;
- производная у квадрата простая — пригодится дальше.

### Геометрическая интуиция

Представьте облако точек на плоскости и прямую, которая через него идёт. Для каждой точки померяйте вертикальное расстояние до прямой. Возведите в квадрат. Сложите. Минимизируйте.

## Как мы ищем $w$ и $b$

Два пути.

### Аналитически

Для линейной регрессии существует **закрытое решение** — формула, которая сразу даёт оптимальные веса. Если объединить веса и сдвиг в матричную форму, ответ выглядит так:

$$\mathbf{w}^* = (X^T X)^{-1} X^T y$$

Это работает, но требует обращения матрицы — для миллионов признаков становится дорого.

### Градиентным спуском

Универсальный подход, который масштабируется на любые модели. Мы шагаем по поверхности ошибки в сторону, противоположную градиенту:

$$w \leftarrow w - \eta \frac{\partial L}{\partial w}$$
$$b \leftarrow b - \eta \frac{\partial L}{\partial b}$$

где $\eta$ — _скорость обучения_ (learning rate).

Производные для MSE:

$$\frac{\partial L}{\partial w} = -\frac{2}{n} \sum_{i=1}^{n} x_i (y_i - \hat{y}_i)$$

$$\frac{\partial L}{\partial b} = -\frac{2}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)$$

## Минимальная реализация на numpy

```python
import numpy as np

def fit(x, y, lr=0.01, epochs=1000):
    w, b = 0.0, 0.0
    n = len(x)
    for _ in range(epochs):
        y_hat = w * x + b
        err = y - y_hat
        dw = -2/n * np.sum(x * err)
        db = -2/n * np.sum(err)
        w -= lr * dw
        b -= lr * db
    return w, b

# example
x = np.array([1, 2, 3, 4, 5])
y = np.array([2.1, 4.0, 6.1, 8.0, 10.2])
w, b = fit(x, y)
print(f"y ≈ {w:.2f}·x + {b:.2f}")
```

После прогона $w \approx 2$, $b \approx 0$ — модель «нашла» закономерность $y \approx 2x$.

## Что важно запомнить

- Модель = семейство функций с параметрами.
- Обучение = поиск параметров, минимизирующих ошибку.
- MSE — частный случай. Для других задач (классификация, ранжирование) функции ошибки другие.
- Градиентный спуск — это рецепт, который работает не только для линий, но и для нейросетей с миллиардами весов. Принцип тот же.
